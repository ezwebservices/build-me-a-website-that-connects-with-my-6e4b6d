# Deploying with AWS Amplify Gen 2

This project is a Vite + React + TypeScript app backed by an Amplify Gen 2 stack
(`amplify/` directory: auth, data models, Lambda functions, storage). The frontend
runs in preview mode until `amplify_outputs.json` is generated by a real deploy.

## Prerequisites

- AWS account with permissions to create Amplify apps, Cognito, DynamoDB, S3, Lambda, SSM.
- The repo pushed to GitHub (or CodeCommit / GitLab / Bitbucket — this guide uses GitHub).
- A Stripe account with a restricted/secret key.
- Node.js **20.20+** and npm **10.8+** locally.

## 1. Connect the repo in the Amplify console

1. Open the [AWS Amplify console](https://console.aws.amazon.com/amplify/).
2. Click **Create new app** → **Host web app** → **GitHub** and authorize AWS Amplify.
3. Pick this repository and the branch you want to deploy (typically `main`).
4. Amplify auto-detects `amplify.yml` at repo root — keep the defaults. The build
   output directory is `dist/`.
5. On the **App settings** step, leave **Enable full-stack deployments** checked. This
   triggers `npx ampx pipeline-deploy` during the build so Gen 2 resources are
   provisioned and `amplify_outputs.json` is written.

## 2. Set backend secrets

The `stripe-checkout` Lambda in `amplify/functions/stripe-checkout/` reads its
secret via `secret('STRIPE_SECRET_KEY')` from `@aws-amplify/backend`, which resolves
from SSM Parameter Store. Set the value for each deployed branch from the Amplify
console: **App settings → Secrets → Manage secrets** (scope it to the branch you
deployed, e.g. `main`). After saving, redeploy the branch so the Lambda picks the
new SSM value up at cold start.

For local sandbox development (optional):

```bash
npx ampx sandbox secret set STRIPE_SECRET_KEY
npx ampx sandbox
```

The sandbox command writes a local `amplify_outputs.json` so the frontend lights up
without a full cloud deploy.

## 3. Frontend environment variables

Amplify Hosting → **App settings → Environment variables** — add any `VITE_*` vars
you want injected into the browser bundle. This project reads:

| Variable                      | Purpose                                               |
| ----------------------------- | ----------------------------------------------------- |
| `VITE_STRIPE_PLAN_NAME`       | Display name of the subscription plan (optional).     |
| `VITE_STRIPE_PLAN_AMOUNT`     | Price in cents (optional).                            |
| `VITE_STRIPE_PLAN_CURRENCY`   | Three-letter currency code (optional, defaults usd).  |
| `VITE_STRIPE_PLAN_INTERVAL`   | `month` / `year` (optional).                          |

## 4. Trigger the first build

Push a commit (or click **Redeploy this version** in the console). The build will:

1. `npm ci` — install dependencies.
2. `npx ampx pipeline-deploy --branch <branch> --app-id <appId>` — provision the
   Gen 2 stack and write `amplify_outputs.json`.
3. `npm run build` — Vite outputs static assets to `dist/`.
4. Amplify Hosting serves `dist/` behind its CDN.

## 5. Expected build output

A successful build logs something like:

```
✔ Deployment completed in ~4m
✔ amplify_outputs.json written
vite v7 building for production...
✓ 1200 modules transformed.
dist/index.html            0.6 kB │ gzip: 0.4 kB
dist/assets/index-xxxx.css 42   kB │ gzip: 8  kB
dist/assets/index-xxxx.js  380  kB │ gzip: 120 kB
✓ built in 9.2s
```

After deploy:

- The hosted URL loads the full site with live product data.
- The `/admin` Studio shows the Cognito sign-in form (first email to register becomes
  the owner — lock down sign-up afterward via `amplify/auth/resource.ts`).
- The `stripe-checkout` Lambda Function URL (visible under Amplify console →
  **Backend environment → Functions → stripe-checkout**) is what the frontend posts
  to when the cart kicks off a Checkout Session.

## 6. Post-deploy smoke test

1. Sign in to `/admin`.
2. Create a product via **Studio → Products → New product**, upload an image.
3. Open the public `/shop` route in an incognito tab and confirm the product appears.
4. Add it to the cart and check out using a Stripe [test card](https://stripe.com/docs/testing) (e.g. `4242 4242 4242 4242`).
5. Confirm the order is recorded in Stripe and that the success route returns the
   buyer to the site.

## Troubleshooting

- **Build succeeds but site still shows Preview mode.** Verify `amplify_outputs.json`
  is committed to the build artifacts — check the Amplify build log for
  `amplify_outputs.json written`. If missing, the full-stack step was skipped; re-enable
  it under **App settings → General → Edit**.
- **Stripe Lambda returns 500.** Confirm `STRIPE_SECRET_KEY` is set for the current
  branch under **App settings → Secrets** and redeploy so the Lambda picks the new
  SSM value up at cold start.
- **Authenticator shows "UserPool not configured".** `amplify_outputs.json` is either
  the placeholder or not bundled — clear the Amplify build cache and redeploy.
